# rdbatch v0.3 — Torrent Search

## Goal

Add a `rdbatch search` command that opens an interactive TUI for finding torrents via a Comet scraper instance. Users search by title (resolved through Cinemeta), drill into a movie or TV episode, see scraped torrents with debrid cache indicators, and either **add** them to the active provider or **stream** them immediately if cached.

This eliminates the need to leave the terminal to find magnets — the existing `fetch` and `list` flows become "I already have the magnet" / "I already added it" paths, while `search` becomes the discovery path.

---

## Scope

### In scope

- New `rdbatch search` command (no arguments — opens an empty search box).
- Title resolution via **Cinemeta** (`https://v3-cinemeta.strem.io`), no API key required.
- Torrent scraping via a user-provided **Comet** manifest URL.
- Search → results → (season → episode for TV) → torrents → action flow.
- Two actions on a torrent: `Enter` to add to the active provider, `W` to stream via `mpv`/VLC (only when cached on the configured Comet debrid).
- Cache indicators in the torrent list.
- Toggle to show/hide uncached torrents.
- Config field + env var for the Comet manifest URL.

### Out of scope

- TMDB as an alternative metadata source.
- Jackett or other Torznab indexers.
- Stream-on-uncached (W is a no-op when uncached — no add-and-wait fallback).
- `rdbatch search "<query>"` shortcut form. Always opens with an empty box.
- Persisting search history across sessions.
- Detecting mismatches between `RDBATCH_PROVIDER` and the debrid service configured in the user's Comet manifest. Document the requirement instead.

## Configuration

New optional field in `~/.config/rdbatch/config.json`:

```json
{
  "provider": "torbox",
  "torbox_api_key": "xxxxxxxx",
  "comet_url": "https://comet.example.com/manifest.json"
}
```

And a new env var (overrides config file):

```bash
export COMET_URL="https://comet.example.com/manifest.json"
```

The URL must point to a full Comet manifest endpoint and must end in `/manifest.json`. Both forms are valid:

- `https://comet.example.com/manifest.json` (default Comet settings)
- `https://comet.example.com/{b64config}/manifest.json` (user-configured Comet, with embedded debrid credentials)

**No API key is required for Cinemeta.** It is a public Stremio metadata addon. Users only need to provide the Comet manifest URL.

### Error: no Comet URL configured

If `rdbatch search` is invoked without `COMET_URL` resolved from either env or config, print to stderr and exit non-zero:

```
error: search requires a Comet manifest URL.

The 'fetch' and 'list' commands work without it. To enable search,
set COMET_URL or add "comet_url" to ~/.config/rdbatch/config.json:

  export COMET_URL="https://your-comet-instance/manifest.json"

The Comet instance must be configured for the same debrid provider
as RDBATCH_PROVIDER. See README.md.
```

## User Flow

### Screen 1 — Search

Opens with the input box focused. Empty results pane below.

```
┌─ Search ─────────────────────────────────┐
│ > the matrix_                            │
└──────────────────────────────────────────┘
  The Matrix (1999) · movie
  The Matrix Reloaded (2003) · movie
  The Matrix Resurrections (2021) · movie
  ...

  [↑/↓ navigate · Enter view torrents · / focus search · Esc quit]
```

- Typing in the input does not fire a query. The query fires only on `Enter` from the input.
- `Enter` in the input → query Cinemeta, replace the results list.
- `↓` from the input → move focus into the results list.
- `↑` from the top result → return focus to the input.
- `/` from the list → return focus to the input (does not clear it).
- `Enter` on a result:
  - `type == "movie"` → go to Screen 3 (torrents).
  - `type == "series"` → fetch full meta, then go to Screen 2a (season picker).
- `Esc` → quit.

**Loading:** while Cinemeta is in flight, show a spinner inline in the input bar (e.g. `> the matrix ⠋`). Keep prior results visible until new ones arrive.

**Empty results:** render a single muted line `no results — try a different query`.

**Network error:** render a single red line `cinemeta error: <message>`. Keep the input editable.

### Screen 2a — Season picker (TV only)

```
Breaking Bad
┌─ Season ────────────────────────────────┐
  Season 1
  Season 2
  Season 3
  Season 4
  Season 5

  [↑/↓ navigate · Enter pick · Esc back]
```

Built by grouping the `meta.videos` array from Cinemeta by `season`. Skip season 0 (specials) unless it is the only season available.

- `Enter` → Screen 2b.
- `Esc` → back to Screen 1, search input and results preserved.

### Screen 2b — Episode picker (TV only)

```
Breaking Bad · Season 1
┌─ Episode ───────────────────────────────┐
  S01E01 · Pilot
  S01E02 · Cat's in the Bag...
  S01E03 · ...And the Bag's in the River
  ...

  [↑/↓ navigate · Enter pick · Esc back]
```

Built by filtering `meta.videos` to the selected season, sorted by `episode` ascending.

- `Enter` → Screen 3.
- `Esc` → back to Screen 2a, season selection preserved.

### Screen 3 — Torrents

```
Breaking Bad · S01E01
┌──────────────────────────────────────────┐
  [⚡] 1080p WEB-DL · 2.1 GB · 47 seeds
  [⚡] 1080p BluRay · 4.3 GB · 112 seeds
  [⬇] 2160p REMUX · 18 GB · 3 seeds
  ...

  [Enter add · W stream · U toggle uncached · Esc back]
```

**Cache indicator:**
- `[⚡]` → cached on the Comet-configured debrid service. Eligible for `W`.
- `[⬇]` → uncached. Hidden by default. Adding via `Enter` works; `W` is a no-op.

**Filtering:** by default the list shows only cached torrents (`[⚡]`). Press `U` to toggle visibility of uncached torrents.

**Sorting:** cached first; within each group, seeders descending then size descending.

**Interactions:**
- `Enter` → reconstruct a magnet from the torrent's info hash and pass it to `provider.AddMagnet(...)`. Show a one-line toast `added · view in 'rdbatch list'` and stay on screen.
- `W` → invoke `internal/player.Play(stream.URL)` (the existing player package). Only enabled when the torrent is cached. On uncached, briefly flash `W disabled — torrent is not cached`.
- `Esc` → back to Screen 2b (TV) or Screen 1 (movie). All upstream state preserved.
- `Q` / `Ctrl+C` → quit.

**Loading:** centered spinner with `scraping torrents...`.

**Empty:** `no torrents found for this title`. `Esc` to go back.

## API Specifications

These are the exact endpoints and response shapes the implementation must use. Do not invent paths or fields — these were verified against live endpoints and the Comet source.

### Cinemeta — title search

**Endpoint (movie):**
```
GET https://v3-cinemeta.strem.io/catalog/movie/top/search={url-encoded query}.json
```

**Endpoint (series):**
```
GET https://v3-cinemeta.strem.io/catalog/series/top/search={url-encoded query}.json
```

Issue both requests in parallel and merge `metas` arrays.

**Response shape:**
```json
{
  "query": "matrix",
  "rank": 0.0,
  "cacheMaxAge": 0,
  "metas": [
    {
      "id": "tt0133093",
      "imdb_id": "tt0133093",
      "type": "movie",
      "name": "The Matrix",
      "poster": "https://...",
      "background": "https://...",
      "releaseInfo": "1999",
      "links": [],
      "behaviorHints": { "defaultVideoId": "tt0133093", "hasScheduledVideos": false }
    }
  ]
}
```

**Fields rdbatch must read:** `id` (use as the IMDB ID for Comet), `type` (`"movie"` or `"series"`), `name`, `releaseInfo` (year string — may be `"1999"` or `"1999-2003"` for series).

**Fields to ignore:** `poster`, `background`, `links`, `behaviorHints`.

No auth headers. No API key.

### Cinemeta — series metadata (for season/episode lists)

Only called when the user selects a series result.

**Endpoint:**
```
GET https://v3-cinemeta.strem.io/meta/series/{imdb_id}.json
```

**Response shape (truncated to relevant fields):**
```json
{
  "meta": {
    "id": "tt0903747",
    "imdb_id": "tt0903747",
    "type": "series",
    "name": "Breaking Bad",
    "videos": [
      {
        "id": "tt0903747:1:1",
        "name": "Pilot",
        "season": 1,
        "episode": 1,
        "number": 1,
        "released": "2008-01-21T05:00:00.000Z",
        "overview": "..."
      }
    ]
  }
}
```

**Fields rdbatch must read:** `meta.name`, `meta.videos[].season`, `meta.videos[].episode`, `meta.videos[].name`.

Build seasons by grouping `videos` on `season`. Build episodes by filtering on the selected season and sorting by `episode` ascending.

Note: some entries have `episode: 0` (sometimes recap specials). Display them, do not filter them out.

### Comet — stream scraping

The user provides a manifest URL ending in `/manifest.json`. Strip that suffix to get the **stream base URL**. All stream requests go against that base.

**Endpoint (movie):**
```
GET {stream_base}/stream/movie/{imdb_id}.json
```

**Endpoint (series):**
```
GET {stream_base}/stream/series/{imdb_id}:{season}:{episode}.json
```

The colon-delimited `id:season:episode` form is the standard Stremio addon protocol, not a Comet-specific quirk.

Example:
- User config: `https://comet.example.com/eyJkZWJyaWQi...==/manifest.json`
- Stream base: `https://comet.example.com/eyJkZWJyaWQi...==`
- Series stream URL: `https://comet.example.com/eyJkZWJyaWQi...==/stream/series/tt0903747:1:1.json`

**Response shape:**
```json
{
  "streams": [
    {
      "name": "[Realdebrid⚡] Comet 1080p",
      "description": "Breaking.Bad.S01E01.1080p.BluRay.x264-GROUP\n💾 2.1 GB · 👥 47 · 📺 WEB-DL",
      "behaviorHints": {
        "bingeGroup": "comet|realdebrid|...",
        "filename": "Breaking.Bad.S01E01.1080p.BluRay.x264-GROUP.mkv",
        "videoSize": 2254857830
      },
      "url": "https://comet.example.com/playback/{info_hash}/{entry_index}/{file_index}/1/1?torrent_name=...&name=...&media_id=..."
    },
    {
      "name": "[Realdebrid🧲] Comet 1080p",
      "description": "...",
      "behaviorHints": { "bingeGroup": "comet|torrent|...", "filename": "...", "videoSize": 1234567890 },
      "infoHash": "abcdef0123456789abcdef0123456789abcdef01"
    }
  ]
}
```

**Field semantics (from Comet's `comet/api/endpoints/stream.py`):**

| Field | Presence | Meaning |
|---|---|---|
| `name` | always | Format: `[{Service}{icon}] Comet {resolution}`. The **icon** indicates state: `⚡` = cached on debrid, `⬇️` = uncached on debrid (Comet will queue it), `🧲` = P2P fallback (no debrid involvement). |
| `description` | always | Multi-line human label. Top line is the raw release title; subsequent lines have size/seeders/quality with emoji prefixes. Display verbatim — do not try to re-parse. |
| `behaviorHints.filename` | usually | Original release file name. |
| `behaviorHints.videoSize` | usually | Size in bytes. Use this for sorting. |
| `url` | when `⚡` or `⬇️` | Comet's playback proxy URL. Path format: `/playback/{info_hash}/{entry_index}/{file_index}/{season}/{episode}?...`. Play directly with mpv/VLC — Comet's proxy resolves and redirects to the actual debrid CDN URL. The info hash is the third path segment. |
| `infoHash` | when `🧲` | The torrent's info hash, for the P2P fallback case where Comet did not consult debrid. |

**Cache state classification rules for rdbatch:**

- Cached: `name` contains the `⚡` rune (U+26A1).
- Uncached debrid: `name` contains the `⬇️` rune (U+2B07 followed by U+FE0F).
- P2P fallback: `name` contains the `🧲` rune (U+1F9F2).

A single stream object belongs to exactly one of these three categories. Treat the P2P fallback as uncached for filtering purposes — it is also not eligible for `W`.

**Info hash extraction for `Enter` (add to provider):**

1. If the stream has a top-level `infoHash` field → use it.
2. Otherwise parse it from the `url` path: the third path segment after `/playback/` is the info hash (40 hex chars).

Reconstruct the magnet as:
```
magnet:?xt=urn:btih:{info_hash}
```

Then call `provider.AddMagnet(magnet)` — no changes needed to the `Provider` interface.

**Streaming URL for `W`:**

For cached streams (`⚡`), pass `stream.url` directly to `internal/player.Play(...)`. mpv and VLC both follow HTTP redirects, so the proxy URL works without resolution on rdbatch's side.

**Error / status streams to skip:**

Comet sometimes returns informational streams instead of real results, identified by a `url` of `https://comet.feels.legal`. Filter these out before rendering. Examples of `name` values to expect on these: `[❌] Comet`, `[⚠️] Comet`, `[🔄] Comet`. The user does not need to see them.

**No auth headers** are sent to Comet — credentials are baked into the `{b64config}` path segment of the manifest URL that the user provides.

## Architecture

### New files

```
internal/
  search/
    cinemeta.go       Cinemeta HTTP client (Search, Meta)
    comet.go          Comet HTTP client (Streams, classify, extract info hash)
    types.go          TitleResult, SeasonGroup, Episode, TorrentResult, CacheState
  ui/
    search.go         Bubble Tea root model (state machine across screens)
    components.go     Reusable input/list/spinner helpers if needed
  commands/
    search.go         Cobra command wiring
```

### Cobra command (`internal/commands/search.go`)

```go
var searchCmd = &cobra.Command{
    Use:   "search",
    Short: "Search for torrents via Comet",
    RunE: func(cmd *cobra.Command, args []string) error {
        cometURL := config.CometURL()
        if cometURL == "" {
            fmt.Fprintln(os.Stderr, noCometURLHelp)
            os.Exit(1)
        }
        cm := search.NewCinemeta()
        co, err := search.NewComet(cometURL)
        if err != nil {
            return err
        }
        return ui.RunSearch(cm, co, provider)
    },
}
```

`provider` is the `Provider` instance already initialized by `commands.SetProvider()`.

### Config (`internal/config/`)

Add a new resolver `CometURL() string` that returns the env var if set, falls back to the config file's `comet_url` field, otherwise empty string.

### Cinemeta client (`internal/search/cinemeta.go`)

Two exported methods:

```go
func (c *Cinemeta) Search(ctx context.Context, query string) ([]TitleResult, error)
func (c *Cinemeta) Meta(ctx context.Context, imdbID string) (Meta, error)
```

`Search` fires both `/catalog/movie/...` and `/catalog/series/...` requests in parallel via `errgroup`, merges results, preserves the order returned by the API (which is already ranked).

`Meta` is only called for series. It hits `/meta/series/{imdb_id}.json`.

`http.Client` with a 10-second timeout. Standard JSON decoding.

### Comet client (`internal/search/comet.go`)

```go
func NewComet(manifestURL string) (*Comet, error)  // strips /manifest.json, errors if absent
func (c *Comet) Streams(ctx context.Context, mediaType, mediaID string) ([]TorrentResult, error)
```

`mediaID` is either `tt1234567` (movie) or `tt1234567:1:1` (series). The caller is responsible for the colon format.

`Streams` performs the GET, decodes into the Stremio stream schema, filters out informational streams (`url == "https://comet.feels.legal"`), classifies each remaining stream's cache state by checking for the `⚡`, `⬇️`, or `🧲` runes in `name`, extracts the info hash via the rules above, and returns `[]TorrentResult`.

`TorrentResult` shape:

```go
type CacheState int
const (
    Cached CacheState = iota   // ⚡
    Uncached                   // ⬇️ or 🧲
)

type TorrentResult struct {
    Name        string      // raw name field, e.g. "[Realdebrid⚡] Comet 1080p"
    Description string      // raw description, multi-line
    InfoHash    string      // 40-char hex
    StreamURL   string      // Comet playback proxy URL, empty if P2P-only
    SizeBytes   int64       // from behaviorHints.videoSize, 0 if absent
    Cache       CacheState
}
```

Comet HTTP timeout: 30 seconds (scraping can be slow on first hit).

### UI state machine (`internal/ui/search.go`)

Single Bubble Tea model with an enum:

```go
type screen int
const (
    screenSearch screen = iota
    screenSeasons
    screenEpisodes
    screenTorrents
)
```

Model fields (preserved across screen transitions so `Esc` keeps state):

- `query string`
- `titleResults []TitleResult`
- `selectedTitle *TitleResult`
- `meta *Meta` (series only)
- `seasons []int`
- `selectedSeason int`
- `episodes []Episode`
- `selectedEpisode *Episode`
- `torrents []TorrentResult`
- `showUncached bool`
- `loading bool`
- `errMsg string`

All HTTP work runs via `tea.Cmd` returning custom messages (`titleResultsMsg`, `metaMsg`, `torrentsMsg`, `errMsg`).

`Esc` decrements `screen`; data above the new screen stays in the model.

## Acceptance Criteria

- [ ] `rdbatch search` without `COMET_URL` prints the configured help message to stderr and exits non-zero.
- [ ] `rdbatch search` with `COMET_URL` opens the search TUI with input focused and an empty results pane.
- [ ] Typing a query and pressing `Enter` populates the results from Cinemeta (both movies and series, merged).
- [ ] Editing the query and pressing `Enter` again replaces the results.
- [ ] `↓` from the input focuses the results list; `↑` from the top result returns focus to the input; `/` from anywhere in the list focuses the input without clearing it.
- [ ] Selecting a movie result goes directly to the torrents screen.
- [ ] Selecting a series result fetches its meta, shows the season picker, then the episode picker, then the torrents screen.
- [ ] `Esc` from any non-root screen returns to the previous screen with state preserved.
- [ ] The torrents screen displays cache indicators (`[⚡]` / `[⬇]`) parsed from the Comet stream `name` field.
- [ ] Uncached torrents are hidden by default; `U` toggles their visibility.
- [ ] Comet informational streams (those with `url == "https://comet.feels.legal"`) are filtered out and never displayed.
- [ ] `Enter` on a torrent reconstructs a magnet from its info hash and calls `provider.AddMagnet`; a confirmation toast appears.
- [ ] `W` on a cached torrent launches `mpv` (or VLC fallback) with the Comet playback URL.
- [ ] `W` on an uncached torrent is a no-op and shows a brief disabled flash.
- [ ] All network failures (Cinemeta unreachable, Comet unreachable, provider error) render an inline error string and leave the TUI usable.
